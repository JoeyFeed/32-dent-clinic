#

План разработки сайта стоматологической клиники "32 Дент"

## Архитектура проекта

Проект будет состоять из трех основных частей:

- **Frontend**: React приложение с современным UI в стиле Arcadia Dental

- **Backend**: Node.js + Express API сервер

- **Database**: PostgreSQL база данных

## Backend API

### Модели данных ([backend/models/](backend/models/))

**User Model** ([backend/models/User.js](backend/models/User.js))

- id, name, phone, email, created_at

**Doctor Model** ([backend/models/Doctor.js](backend/models/Doctor.js))

- id, name, specialty, experience, photo_url, bio

**Service Model** ([backend/models/Service.js](backend/models/Service.js))

- id, name, category, description, price_from, price_to

**Appointment Model** ([backend/models/Appointment.js](backend/models/Appointment.js))

- id, user_id, doctor_id, service_id, date, time, status

**Review Model** ([backend/models/Review.js](backend/models/Review.js))

- id, user_id, rating, comment, created_at

**Settings Model** ([backend/models/Settings.js](backend/models/Settings.js))

- id, phone, address, hours, social_links

### API Routes ([backend/routes/](backend/routes/))

**Appointments API** ([backend/routes/appointments.js](backend/routes/appointments.js))

- POST /api/appointments - создать запись на приём

- GET /api/appointments - получить записи (admin)

**Doctors API** ([backend/routes/doctors.js](backend/routes/doctors.js))

- GET /api/doctors - список врачей

**Services API** ([backend/routes/services.js](backend/routes/services.js))

- GET /api/services - список услуг и цены

**Reviews API** ([backend/routes/reviews.js](backend/routes/reviews.js))

- GET /api/reviews - отзывы пациентов

- POST /api/reviews - создать отзыв

**Feedback API** ([backend/routes/feedback.js](backend/routes/feedback.js))

- POST /api/feedback - отправить форму обратной связи

### Middleware ([backend/middleware/](backend/middleware/))

- Валидация данных (express-validator)

- Обработка ошибок
- CORS настройка

- Логирование запросов

## Этапы реализации

1. **Настройка проекта**: создание структуры, установка зависимостей

2. **Frontend базовая структура**: компоненты, роутинг, стили

3. **Backend API**: модели, routes, controllers

4. **База данных**: миграции, seed данные

5. **Интеграция**: подключение frontend к backend

6. **Тестирование**: функциональное тестирование, адаптивность
